Introducción

Este Trabajo de Fin de Grado (TFG) se enfoca en el desarrollo de un sistema informático, aplicando buenas prácticas de desarrollo de software y metodologías ágiles. A lo largo del proyecto, se abordaron diversas fases, desde la planificación y análisis hasta la implementación y evaluación, con el objetivo de crear un sistema eficiente y adaptable.

El proyecto establece el contexto y los objetivos, tanto en términos técnicos como académicos. Los objetivos técnicos incluyen la creación de un sistema que mejore la eficiencia y calidad del desarrollo de software, mientras que los objetivos académicos se centran en la aplicación práctica de conocimientos teóricos adquiridos durante la carrera.

La metodología ágil fue adoptada para el desarrollo del proyecto, permitiendo una gestión flexible y dinámica a través de sprints. Se utilizaron herramientas como GitHub, TogglTrack, Google Calendar y Google Chat para una organización eficiente y un seguimiento detallado del progreso.

Se realizó una auditoría de las buenas prácticas del equipo, enfocándose en Bluejay y las TPAs (Team Performance Assessments), incluyendo una exploración de visualizaciones en Grafana y un manual para la creación de widgets personalizados.

El análisis del sistema define el objetivo y describe su arquitectura y diseño funcional, abordando tanto el frontend como el backend y proporcionando una visión completa de su diseño y funcionalidad.

La implementación del sistema se describe detalladamente, incluyendo el entorno de desarrollo y el stack tecnológico utilizado, así como detalles específicos de la implementación como las traducciones, la gestión de archivos locales y la ejecución de casos de prueba.

Se incluyen manuales para el despliegue y uso del sistema, ofreciendo instrucciones detalladas para el despliegue en modo desarrollo, producción y aplicación, así como una guía de referencia para el archivo de configuración y un tutorial sobre el ciclo de creación de un TPA.

El documento concluye con un análisis de los problemas encontrados durante el desarrollo y propone soluciones y trabajos futuros, destacando la importancia de las metodologías ágiles en el desarrollo de software y el valor de la aplicación práctica de conocimientos teóricos.

En resumen, se proporciona una visión detallada y estructurada del desarrollo de un proyecto de software, desde la introducción y la metodología hasta la implementación y los manuales de uso, concluyendo con un análisis de problemas y propuestas para futuros trabajos.

Estructura del proyecto

La aplicación se divide en dos partes principales: el frontend desarrollado en Angular y el backend en Express. Ambos están diseñados para ser modulares, escalables y mantener una estructura clara. Además, el frontend interactúa con la API propia (backend), con la API de GitHub y con la API de Bluejay, lo que permite una amplia gama de funcionalidades y una integración eficiente con servicios externos.

Frontend

  • Modelos: Definen la estructura de los datos para tipar respuestas y peticiones HTTP.
  • Servicios: Encargados de la comunicación con el backend y APIs externas, incluyendo servicios para repositorios, TPAs, Bluejay y GitHub.
  • Componentes: Bloques reutilizables de UI para funcionalidades específicas como gestión de TPAs, panel de métricas, integración con GitHub, y más.
  • Páginas: Vistas completas que agrupan componentes para ofrecer funcionalidades principales como gestión de TPAs, visualización de métricas, y tests con GitHub.

Backend

  • Gestión de Repositorios: Incluye clonado, gestión de archivos y ejecución de scripts.
  • Manejo de Métricas, Scripts y TPAs: Operaciones CRUD y gestión de TPAs.
  • Documentación API: Utilización de Swagger para documentar la API, facilitando su uso y pruebas.

Esta estructura permite una interacción eficiente entre el frontend y el backend, asegurando una gestión adecuada de métricas, TPAs y repositorios, además de facilitar la integración con APIs externas.

Estructura de las páginas

Página de Inicio

Al iniciar TP-Tester, la página inicial es "Home". Esta página contiene una organización por secciones del contenido equivalente al archivo Readme del proyecto, describiendo el funcionamiento de cada página. En la parte izquierda, se puede navegar a todas las secciones disponibles de la documentación, y en la parte superior derecha, se puede seleccionar el idioma de visualización entre español e inglés.

Página Testear Métricas

Esta página está diseñada para gestionar y probar métricas, proporcionando una interfaz amigable para visualizar, crear y gestionar métricas. Se accede desde cualquier página haciendo clic en “Metrics”.

Estructura de la Página

La página muestra todas las métricas guardadas en dos grandes tablas: una para métricas relacionadas con el TPA y otra para métricas individuales no relacionadas con ningún TPA específico. Cada fila de la tabla incluye el nombre de la métrica y botones para ver, ejecutar/editar, y eliminar la métrica. Además, hay una sección para crear nuevas métricas introduciendo el contenido en un área de texto y guardándolas como JSON.

Testeo Manual

Esta página permite clonar nuevos repositorios o trabajar con repositorios ya clonados. Al entrar, se pueden ver todos los repositorios accesibles mediante un token, y cada repositorio tiene botones para ver detalles y editar.

Clonar Nuevos Repositorios

Se pueden ver los repositorios accesibles con el token introducido, mostrando detalles del repositorio y permitiendo clonar el repositorio para realizar acciones sobre él en la pantalla de repositorios clonados.

Repositorios Clonados

Permite trabajar con repositorios clonados previamente, proporcionando una descripción breve y un botón para comenzar a trabajar con el repositorio clonado. Incluye opciones para gestionar ramas (ver, crear, eliminar, cambiar), pull requests (crear, ver, fusionar) y otras acciones generales en el repositorio.

Página de Testing Automático

Esta sección proporciona una interfaz para gestionar scripts YAML guardados y ejecutar pruebas automáticas. Ofrece una zona de ejecución donde se pueden gestionar y ejecutar scripts de prueba, así como visualizar resultados.

Página Testeo de TPAs

Esta página está dedicada a probar TPAs específicos, permitiendo la creación, edición y ejecución de pruebas de TPA. Incluye herramientas para gestionar y analizar los resultados de las pruebas.

Página de Configuración

Esta página permite configurar diversos aspectos del sistema, incluyendo la gestión de tokens para acceder a repositorios y otras configuraciones necesarias para el funcionamiento del sistema TP-Tester.

Este apartado proporciona un manual detallado para usuarios, permitiendo una comprensión completa y un uso eficiente del sistema TP-Tester, cubriendo desde la página de inicio hasta la configuración y ejecución de pruebas automáticas y manuales.
